### Learning 1 — Install-path vs repo-path divergence

- **Type:** gotcha
- **Domain:** process
- **Priority:** critical
- **Applicability:** 9 (current project)
- **General Applicability:** 8 (any project with postinstall + global skills)
- **Tags:** [process, deployment, postinstall, npm, skills-directory, critical, intermediate]
- **Title:** "Edit the repo path; the install path is read-only after install"
- **Problem:** The agent made all 5 SKILL.md edits for plan 2026-06-29-003 in `~/.agents/skills/pwrl-*/` (the install path) instead of `~/Projects/pwrl/pwrl-*/` (the repo path). The next `npm install` will overwrite the install, and the published package was never updated.
- **Application:** Before editing any file in a `pwrl-*/` directory, verify the tree. The repo at `${REPO_ROOT}/pwrl-*/` is the source of truth; `~/.agents/skills/pwrl-*/` is a deployment artifact. All `files:` paths in task frontmatter must use the repo path.
- **Related Learnings:**
  - [complements] `cross-skill-contract-enforcement-2026-06-29.md` — extends the contract pattern to the install-vs-repo layer
  - [extends] `task-state-machine-enforcement-2026-06-29.md` — adds the location dimension to ownership

### Learning 2 — Code-edit Pre-Flight Guard gap

- **Type:** pattern
- **Domain:** architecture
- **Priority:** important
- **Applicability:** 9 (current project)
- **General Applicability:** 7 (applies to any framework with multi-tree deployment)
- **Tags:** [architecture, contract-enforcement, pre-flight-guard, multi-tree, intermediate]
- **Title:** "Extend the Pre-Flight Guard contract to code-edit location"
- **Problem:** The U1–U5 contract (in `pwrl-work/SKILL.md` §"Task Lifecycle Contract") defines who owns each task file transition. It does not define who owns the editing of source code files. The agent treated code edits as "unowned" and edited the wrong tree.
- **Application:** Extend the contract to specify not just **who** performs a transition but **where** it takes place. For PWRL, add a "Code-Edit Location" rule: "All edits to `pwrl-*/SKILL.md` must target the repo tree, never the install tree." Encode as a Pre-Flight Guard at the top of `pwrl-work/SKILL.md` (or a sibling).
- **Related Learnings:**
  - [extends] `task-state-machine-enforcement-2026-06-29.md`
  - [extends] `cross-skill-contract-enforcement-2026-06-29.md`
